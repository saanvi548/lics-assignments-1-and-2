Task 2: First Negative in Array

Formal Specifications (from task2.dfy)
Precondition: requires a.Length >= 0
Postconditions:
  • ensures -1 <= index < a.Length (Output is a valid index or -1)
  • ensures index == -1 ==> (forall k :: 0 <= k < a.Length ==> a[k] >= 0) (If -1, all elements are non-negative)
  • ensures index >= 0 ==> a[index] < 0 && (forall k :: 0 <= k < index ==> a[k] >= 0) (If found, it is negative and all preceding elements are non-negative)
Loop Invariants:
  • invariant 0 <= i <= a.Length (Loop bounds)
  • invariant (forall k :: 0 <= k < i ==> a[k] >= 0) (All elements examined so far are non-negative)
Loop Variant: decreases a.Length - i (Ensures termination)

Experimental Log

Experiment Observation Result
Missing Invariant Omitted the forall invariant. Dafny could not prove that the element found (if any) was the *first* negative one. Verification Failed
Final Logic The forall invariant establishes the crucial "history" property, linking the current loop state to the required postcondition of minimality. ✔ Verified

Task 4: Tribonacci Sequence

Formal Specifications (from task4.dfy)
Precondition: requires n >= 0
Postcondition: ensures result == T(n) (Correctness condition)
Loop Invariants:
  • invariant 3 <= i <= n + 1 (Loop bounds)
  • invariant a == T(i - 3) (Tracks the value three steps back)
  • invariant b == T(i - 2) (Tracks the value two steps back)
  • invariant c == T(i - 1) (Tracks the value one step back)
  • invariant result == c (Relates the final computed variable)
Loop Variant: decreases n - i + 1 (Ensures termination)

Experimental Log

Experiment Observation Result
Incomplete State Tracking Only provided invariants for `c` and `i`. Dafny could not prove that the next value (`a+b+c`) correctly computes `T(i)`. Verification Failed
Final Logic We need to explicitly track the history of the last three states (`a`, `b`, `c`) using separate invariants to enable Dafny to verify the rolling update logic. ✔ Verified
